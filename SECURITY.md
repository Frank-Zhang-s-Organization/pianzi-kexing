# SECURITY

## 安全与合规问题如何反馈

如果你发现的问题涉及以下任一类型，请不要直接公开提交可利用细节：

- 数据来源合法性缺陷
- 可能导致对具体自然人自动指认的实现风险
- 生物特征信息处理风险
- 误报、撤销、审计链设计缺陷
- 任何可能让项目偏离 `docs/COMPLIANCE_BOUNDARY.md` 的代码路径

建议做法：

1. 先通过仓库维护入口联系
2. 用最小必要信息描述问题
3. 避免公开贴出可直接滥用的 PoC

## 项目当前安全边界

这个仓库当前不接受以下方向的贡献：

- 真实人脸模板或向量库
- 非官方“骗子数据库”
- 群众曝光源聚合
- 面向公共空间的实时识别实现

如果某个 PR 试图引入这些能力，维护者会直接拒绝。

## 响应原则

- 合规边界问题优先于功能需求
- 可审计性优先于演示效果
- 默认保守，必要时关闭能力而不是放宽门槛
