# SUPPORT

## 先看哪里

在提问前，建议先阅读：

- `README.md`
- `docs/PRODUCT_VISION.md`
- `docs/COMPLIANCE_BOUNDARY.md`
- `docs/TODO.md`

## 适合提 issue 的内容

- 文档不清楚
- schema / policy / audit 设计建议
- 示例数据结构改进建议
- mock 演示层的架构建议

## 不适合提 issue 的内容

- 要求加入真实识别能力
- 要求接入民间曝光库
- 要求实现公共空间实时识别
- 要求规避法域、授权或审计约束

## 支持边界

当前项目处于研究骨架阶段。

维护者更可能优先处理：

- 文档边界
- 数据结构
- 策略判断
- 审计与治理层

而不是产品化交付问题。
