# 产品愿景

## 一句话定义

`骗子克星` 不是一个面向公共空间的实时抓人系统，而是一个“合规门禁 + 风险清单治理 + 提示策略研究”的本地研究项目。

## 想解决的问题

如果某类高风险名单确实来自官方公开通报，且存在明确法律依据、更新机制和人工复核流程，那么项目需要回答的核心问题是：

- 哪些数据根本不该被纳入？
- 哪些提示场景根本不该发生？
- 即使在合法边界内，系统也该如何默认保守、可审计、可撤销？

## 非目标

- 不做公共空间无差别识别
- 不做基于社交媒体曝光帖的“骗子库”
- 不做自动定罪式 UI
- 不做躲避监管、躲避授权的落地方案
- 不做对未成年人或普通路人的识别方案

## 目标用户

- 做研究原型的开发者
- 需要评估此类方案风险的产品或合规团队
- 关注误报治理、审计日志和数据合法性的安全研究者

## 成功标准

### S1. 治理优先

项目先把数据、权限、提示和撤销机制定义清楚，再讨论识别能力。

### S2. 默认保守

任何来源不明、无法撤销、无法审计、无法说明法域依据的数据，一律拒绝。

### S3. 人工复核优先

系统输出应是“需要复核的风险提示”，而不是“自动断言此人有罪”。
