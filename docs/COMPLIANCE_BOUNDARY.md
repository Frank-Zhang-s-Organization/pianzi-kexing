# 合规边界

## 为什么这个项目要严格收缩

“面部识别 + 风险人群提醒”不是普通功能型开源项目。它天然涉及：

- 生物特征信息处理
- 对具体自然人的识别与标签化
- 误报造成的名誉、歧视和现实冲突
- 对维护者的反向报复风险

因此，本仓库只接受“研究骨架”定位，不接受直接落地的监控型实现。

## 仓库允许的内容

- 官方公开通报名单的元数据结构定义
- 数据来源合法性检查
- 本地化、离线化、显式授权前提下的架构讨论
- 误报治理、撤销机制、审计日志和法律评估文档

## 仓库不接受的内容

- 基于网络曝光帖、论坛帖子、社交媒体爆料的“骗子库”
- 对普通人进行实时识别、跟踪、比对的可执行代码
- 针对儿童、未成年人、公共空间路人的识别方案
- 绕过授权、绕过提示、隐蔽采集的人像抓取方案
- 激化式命名、羞辱性展示或煽动围堵的交互设计

## 数据源门槛

一个条目若要进入清单，至少必须满足：

- 来源是官方公开页面或具有明确法定公开依据
- 可以提供稳定 URL
- 可以记录发布日期
- 可以记录法域和法律依据
- 可以说明撤销或更新机制

以下来源默认拒绝：

- “骗子数据库”
- 社媒曝光合集
- 民间黑名单
- 未经核验的转发截图
- 不可追溯来源的聚合库

## 发布门禁

只有在下面条件都满足时，才值得继续做下一阶段：

1. 已完成律师评估。
2. 已明确适用法域。
3. 已有误报申诉和撤销流程。
4. 已明确项目维护者匿名化和组织化发布方式。
5. 已通过内部伦理审查。
