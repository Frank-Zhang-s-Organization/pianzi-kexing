# 架构草图

## 目标

为一个高风险方向提供“先审查、后实现”的架构骨架。

## 模块划分

### 1. Manifest Ingestion

读取风险清单元数据，不读取真实人脸特征库。

### 2. Policy Gate

对每个清单条目做以下判断：

- 来源是否官方
- 是否具备法域与法律依据
- 是否存在撤销机制
- 是否属于仓库禁止的数据类别

### 3. Matcher Adapter

只保留接口，不提供实现。任何真实识别模型都必须在法律评估后单独开发。

### 4. Alert Adapter

只定义提示等级和文案结构，不提供面向现实人物的实时识别提醒实现。

### 5. Audit Log

记录策略判断结果、版本、来源和人工复核状态。

## 建议数据流

1. 导入清单文件。
2. 运行策略校验。
3. 输出可接受 / 不可接受条目和原因。
4. 仅在合规门禁通过后，才讨论下一阶段。
