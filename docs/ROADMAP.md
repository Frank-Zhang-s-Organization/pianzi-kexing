# 技术路线图

## Phase 0: 治理与发布准备

- 完成维护者去风险化
- 明确项目定位、非目标和拒绝清单
- 固化项目级 AGENTS 和开发蓝图

## Phase 1: 清单治理内核

- 定义 manifest schema
- 完成来源合法性校验器
- 输出条目级拒绝原因和审计摘要
- 增加快照版本号和撤销状态支持

## Phase 2: 策略与审计

- 引入多级策略规则
- 增加人工复核字段
- 设计风险等级与提示等级映射
- 输出审计日志与变更记录

## Phase 3: 适配器接口

- 设计 matcher adapter 抽象接口
- 设计 alert adapter 抽象接口
- 默认提供 mock 实现，不接真实识别引擎
- 通过 feature flag 明确禁用高风险能力

## Phase 4: 实验室原型

前提：

- 只在受控环境
- 只处理获得授权的数据
- 只做人机协同演示

交付：

- 离线演示模式
- 人工确认后的提示 UI
- 全量审计记录

## 暂不进入的阶段

- 公共空间部署
- 面向陌生人的实时识别
- 民间数据库聚合
